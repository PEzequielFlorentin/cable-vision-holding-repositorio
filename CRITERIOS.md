# Desarrollo de Aplicación con Java

Trabajo Práctico Final

## Tema

El trabajo práctico final consistirá en el desarrollo de una aplicación
completa utilizando **Java** y una **base de datos relacional**, basada
en el mismo tema del primer trabajo práctico del año.\
Si el grupo desea cambiar el tema, deberá **consultarlo previamente** y
presentar una propuesta clara y fundamentada.

## Requisitos Técnicos

### Actores (Usuarios del Sistema)

El sistema deberá contar con **al menos 3 actores distintos**, cada uno
con funcionalidades diferenciadas.

### Frameworks

Se deberán utilizar **mínimo 2 frameworks** de Java, tales como:\
- Spring Boot / Spring MVC / Spring Data JPA\
- Hibernate\
- Thymeleaf\
- (u otros frameworks equivalentes, con aprobación del docente)

### Base de Datos

-   Deberá ser **relacional** (MySQL, PostgreSQL, H2, etc.).\
-   Se debe diseñar el **modelo entidad-relación**.\
-   La aplicación deberá incluir **persistencia completa** (ABM
    funcional).

### Arquitectura del Proyecto

El proyecto debe estar organizado por capas:\
- Controlador\
- Servicio\
- Repositorio\
- Modelo

### Control de Versiones

-   El proyecto deberá estar alojado en **GitHub**.\
-   Todos los integrantes deben aparecer como **colaboradores activos**.

## Evaluación Documentada

La entrega deberá incluir un documento PDF con la siguiente estructura:

### Portada

-   Nombre de la materia\
-   Docente\
-   Integrantes del grupo\
-   Curso y año\
-   Título del proyecto\
-   Link al repositorio de GitHub (con proyecto completo y ejecutable)

### Descripción General del Proyecto

-   Tema elegido\
-   Objetivos del sistema\
-   Descripción de los actores y sus roles

### Arquitectura y Desarrollo

-   Descripción de las capas del sistema\
-   Explicación de la estructura del proyecto\
-   Tecnologías y frameworks utilizados\
-   Descripción de las clases principales y responsabilidades\
-   Capturas o ejemplos de código relevantes

### Casos de Uso y Diagramas

-   Diagrama de casos de uso con actores\
-   Breve descripción de cada caso de uso

### Base de Datos

-   Diagrama entidad-relación\
-   Descripción de tablas y relaciones\
-   Ejemplo de datos cargados (opcional)

### Manual de Usuario

Debe incluir instrucciones para cada actor del sistema:\
- Cómo acceder al sistema\
- Funcionalidades disponibles\
- Ejemplos y capturas de pantalla

## Criterios de Evaluación

-   Cumplimiento de los requisitos técnicos\
-   Aplicación de frameworks\
-   Organización y estructura del código\
-   Claridad de la documentación\
-   Diseño de base de datos\
-   Funcionamiento completo\
-   Interfaz clara\
-   Trabajo colaborativo y uso correcto de GitHub\
-   Defensa oral individual
